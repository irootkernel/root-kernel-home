Hermes Agent Supports

Dozens of Kkachi Agents separated by project and role

Root Kernel uses Hermes Agent as its main Agent runtime and operates roles, memory, research, and review flows through KAO, KLM, and KRQ.

Dozens of project-scoped and role-scoped Kkachi Agents work by project and role, splitting one human-defined goal into role-specific work pieces and completing it together.

Operating Evidence

A real operating system for running AI like a team

Root Kernel does not use AI Agents only as tools summoned on demand. It separates projects, roles, memory, and review responsibility, then connects each Agent's work piece into one finished deliverable.

Operating principleHow it worksWhy it matters
Role separationBlue, Red, Orange, Grey, Teal, Green, Yellow, and other Agents are separated by responsibility.Execution, review, user perspective, documentation, and research are not mixed into one Agent.
Project separationAgents are operated as project-scoped roles with separated context and work boundaries.This reduces context and memory contamination while keeping each Agent focused on its scope.
Role-based reviewWork is checked again through execution, review, user perspective, and documentation criteria.AI work does not end as a one-off answer. It is shaped into a deliverable through multiple viewpoints.

Kkachi Research Queue

A research and knowledge operations queue where Black and Yellow Kkachi Agents collect, reframe, store, and index knowledge.

Delegation

Delegating work to an Agent organization

Root Kernel does not try to automate everything from the start. A human keeps direction, approval, and final judgment while gradually delegating repeated development, review, documentation, research, and usability checks to role-based Kkachi Agents.

StageWork patternHuman role
Human-centered workRequirements, development, review, documentation, research, and usability checks are concentrated in one person.Execution volume and review depth are constrained by human bandwidth.
After KAO operationBlue executes, Red reviews critically, Orange checks user perspective, Grey keeps documents aligned, and Yellow strengthens research background.The human compares Agent outputs and adjusts direction and priorities.
Human-in-the-loop delegationAgents perform work while final approval, key product judgment, and validation criteria remain human-led.The human focuses more on direction, final approval, and important product decisions.
