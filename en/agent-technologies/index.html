Agent Technologies

AI Agents that deliberate, challenge, and write code

While building its own AI Persona Synthesis Research system, Root Kernel has operated a workflow where people and AI Agents communicate in shared documents, multiple Agents deliberate, and product and server changes move through specification and validation steps.

Doksuri and ATN support AI Agent communication and decision-making across company work. AI-SPARK is the patent-pending technology used specifically in Sudal service server development.

Operating Model

AI collaboration embedded in product development

Agent Technologies are not separate demos. They are working practices accumulated while building Root Kernel's AI Persona Synthesis Research system. Doksuri is the shared document surface for working with AI Agents, and ATN is a moderator-led turn structure that makes Agents share and challenge each other's views.

In Practice

How the operating model works

These technologies are not forced into one linear pipeline. Each one supports a different working situation inside the company.

Working situationHow it is usedWhy it matters
Working through documentsDoksuri keeps requirements, task instructions, results, evidence, and reviews in the same document surface.AI work does not disappear into chat logs. It becomes part of the company's work record.
Running turn-based deliberationATN uses a moderator to assign turns and make Agents see and challenge each other's claims.Instead of a final synthesized answer, the team can inspect who said what, in what order, and how claims were challenged.
Validating server changesAI-SPARK was used in Sudal server development to connect natural-language requirements to Spec Bundle, Validator, and Approval Gate steps.AI-generated server changes are treated as reviewable, validated revisions instead of being applied directly.
