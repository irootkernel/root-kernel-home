Hermes Agent Supports

프로젝트와 역할로 나뉘어 일하는 수십 개의 Kkachi Agent

Root Kernel은 Hermes Agent를 주력 Agent runtime으로 사용하며, KAO, KLM, KRQ를 통해 Agent의 역할, 기억, 연구, 검토 흐름을 운영합니다.

수십 개의 project-scoped, role-scoped Kkachi Agent가 프로젝트와 역할에 따라 분리되어 일하고, 사람이 정한 목표를 역할별 작업 조각으로 나누어 완성합니다.

Operating Evidence

Operating AI Agents Like a Team

Root Kernel은 AI Agent를 필요할 때마다 부르는 도구로만 사용하지 않습니다. 프로젝트, 역할, 기억, 검토 책임을 나누고, 각 Agent가 맡은 작업 조각을 이어 하나의 결과물로 완성하는 운영 구조를 만들고 있습니다.

운영 원칙어떻게 운영되는가왜 중요한가
역할 분리Blue, Red, Orange, Grey, Teal, Green, Yellow 등 Agent가 역할별로 나뉩니다.실행, 리뷰, 사용자 관점, 문서 정리, 조사처럼 서로 다른 책임을 한 Agent에게 섞지 않습니다.
프로젝트 분리Agent는 project-scoped로 운영되어 프로젝트별 맥락과 작업 범위를 분리합니다.프로젝트 context와 memory가 섞이는 문제를 줄이고, 각 Agent가 자신의 작업 범위에 집중하게 합니다.
역할별 검토작업 결과는 실행, 리뷰, 사용자 관점, 문서 정합성 같은 역할 기준으로 다시 확인됩니다.AI 작업이 한 번의 응답으로 끝나지 않고, 여러 관점의 검토를 거쳐 결과물로 정리됩니다.

Technologies

Hermes-based AI Agent Operations

수십 개의 project-scoped, role-scoped Kkachi Agent가 프로젝트와 역할에 따라 분리되어 일하고, 사람이 정한 목표를 역할별 작업 조각으로 나누어 완성합니다.

운영중

KAO

Kkachi Agent Organization

Hermes 위에서 수십 개의 project-scoped, role-scoped Kkachi Agent를 운영하는 Agent Operating Model입니다.

운영중

KLM

Kkachi Letta Memory

Hermes Agent가 Letta memory와 LLM Wiki를 더 정확하게 활용하도록 돕는 plugin, daemon, CLI 프로젝트입니다.

운영중

KRQ

Kkachi Research Queue

Black과 Yellow Kkachi Agent가 자료 조사와 지식 재구성을 통해 검증된 지식 카드를 만들고, 다른 Agent가 재사용하게 하는 연구와 지식 운영 큐입니다.

KAO

Color-based Agent Roles and Responsibilities

KAO의 역할 체계는 color role로 구분됩니다.

Delegation

Delegating Work to Agent Organizations

Root Kernel은 처음부터 모든 일을 자동화하려 하지 않습니다. 사람이 방향, 승인, 최종 판단을 유지한 채 반복적인 개발, 리뷰, 문서 정리, 조사, 사용성 검토를 역할 기반 Kkachi Agent에게 단계적으로 위임하고 있습니다.

단계작업 방식사람에게 남는 역할
사람 중심 작업요구사항 정리, 개발, 리뷰, 문서화, 조사, 사용성 검토가 한 사람에게 집중됩니다.모든 세부 작업을 직접 처리해야 하므로 실행량과 검토 깊이에 한계가 생깁니다.
KAO 운영 후Blue는 실행, Red는 비판적 리뷰, Orange는 사용자 관점, Grey는 문서 정리, Yellow는 조사 배경을 맡습니다.사람은 Agent별 결과를 비교하고, 방향과 우선순위를 조정할 수 있습니다.
Human-in-the-loop 위임Agent가 작업을 수행하되 최종 승인, 중요한 제품 판단, 검증 기준은 사람이 유지합니다.사람은 방향 설정, 최종 승인, 중요한 제품 판단에 더 집중합니다.
