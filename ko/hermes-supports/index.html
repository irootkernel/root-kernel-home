Hermes Agent Supports

프로젝트와 역할별로 운영되는 Kkachi Agent 조직

Root Kernel은 Hermes Agent를 기반으로 KAO, KLM, KRQ를 운영하며, Agent의 역할 분담과 기억 관리, 연구, 검토 흐름을 체계화합니다.

수십 개의 Kkachi Agent가 프로젝트와 역할에 따라 각기 다른 작업을 맡고, 사람이 정한 목표를 여러 작업 단위로 나누어 완성합니다.

Operating Evidence

Operating AI Agents Like a Team

Root Kernel은 AI Agent를 단순 호출형 도구가 아니라, 역할과 책임을 가진 작업 단위로 운영합니다. 프로젝트, 역할, 기억, 검토 책임을 나누고, 각 Agent가 맡은 작업을 이어 하나의 결과물로 완성하는 운영 구조를 만들고 있습니다.

운영 원칙어떻게 운영되는가왜 중요한가
역할 분리Blue, Red, Orange, Grey, Teal, Green, Yellow 등 색상 역할에 따라 Agent를 나누어 운영합니다.실행, 리뷰, 사용자 관점, 문서 정리, 조사처럼 서로 다른 책임을 하나의 Agent에게 몰아주지 않습니다.
프로젝트 분리Agent는 프로젝트별로 분리되어 운영되며, 각 프로젝트의 맥락과 작업 범위를 따로 유지합니다.프로젝트마다 context와 memory를 분리해, 각 Agent가 맡은 작업에 집중할 수 있게 합니다.
역할별 검토작업 결과는 실행, 리뷰, 사용자 관점, 문서 일관성 등 여러 역할의 관점에서 다시 검토합니다.Agent의 작업은 한 번의 응답으로 끝나지 않고, 여러 관점의 검토를 거쳐 결과물로 정리됩니다.

Technologies

Hermes-based AI Agent Operations

수십 개의 Kkachi Agent가 프로젝트와 역할에 따라 각기 다른 작업을 맡고, 사람이 정한 목표를 여러 작업 단위로 나누어 완성합니다.

운영중

KAO

Kkachi Agent Organization

Hermes를 기반으로 여러 Kkachi Agent를 프로젝트와 역할별로 나누어 운영하는 Agent Operating Model입니다.

운영중

KLM

Kkachi Letta Memory

Hermes Agent가 Letta memory와 LLM Wiki를 더 정확하게 활용할 수 있도록 돕는 plugin, daemon, CLI 프로젝트입니다.

운영중

KRQ

Kkachi Research Queue

Black과 Yellow Kkachi Agent가 자료 조사와 지식 재구성을 맡아 검증된 지식 카드를 만들고, 이를 다른 Agent가 재사용할 수 있도록 관리하는 연구·지식 운영 시스템입니다.

KAO

Color-based Agent Roles and Responsibilities

KAO는 색상별 역할 체계로 Agent의 책임을 나눕니다.

ColorRoleDescription
BlueProject Leader사용자와 소통하며 프로젝트 실행을 총괄하고 대부분의 작업을 수행합니다.
RedRed Team ReviewerBlue의 작업물을 비판적으로 검토하고 판단 기준을 보완합니다.
OrangeUser Advocate결과물이 사용자 관점에서 사용성이 좋은지 판단합니다.

TealUX/UI DesignerUX/UI가 필요한 프로젝트에서 디자인 방향을 검토합니다.
GreenBusiness Strategist사업계획서, 마케팅 전략, 시장 전략을 담당합니다.

Delegation

Delegating Work to Agent Organizations

Root Kernel은 모든 일을 한 번에 자동화하기보다, 사람이 방향 설정과 승인, 최종 판단을 유지한 채 반복적인 개발, 리뷰, 문서 정리, 조사, 사용성 검토를 역할 기반 Kkachi Agent에게 단계적으로 위임합니다.

단계작업 방식사람에게 남는 역할
사람 중심 작업요구사항 정리부터 개발, 리뷰, 문서화, 조사, 사용성 검토까지 한 사람이 대부분의 작업을 맡습니다.모든 세부 작업을 직접 처리해야 하므로, 실행 속도와 검토 깊이에 한계가 생깁니다.
KAO 운영 후Blue는 실행을 맡고, Red는 비판적으로 검토하며, Orange는 사용자 관점, Grey는 문서 정리, Yellow는 조사 배경을 담당합니다.사람은 Agent별 결과를 비교하며, 방향과 우선순위를 조정합니다.
Human-in-the-loop 위임Agent가 작업을 수행하되 최종 승인, 중요한 제품 판단, 검증 기준은 사람이 유지합니다.사람은 방향 설정과 최종 승인, 중요한 제품 판단에 집중합니다.
